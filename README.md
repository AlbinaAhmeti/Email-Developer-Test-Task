This project contains a responsive promotional email template built with **HTML** and **inline CSS only**, following email-safe best practices for broad client compatibility.

The email template was created based on the following requirements:

  - Apple Mail

### 1. How would you test this email across different clients and devices?

I would test the email in multiple ways:

- Open it in real inboxes such as **Gmail, Outlook, and Apple Mail**
- Check both **desktop and mobile** rendering
- Use specialized email testing tools such as:
  - **Litmus**
  - **Email on Acid**
- Validate:
  - layout consistency
  - font rendering
  - image display
  - button behavior
  - spacing/padding issues
  - dark mode behavior where relevant

I would also send test campaigns to several devices and inboxes to confirm the final rendering before production use.

### 2. What steps would you take to improve deliverability and reduce the chance of landing in spam?

To improve deliverability, I would focus on both technical setup and email content quality:

- Make sure **SPF, DKIM, and DMARC** are properly configured
- Use a trusted sending domain with a good reputation
- Keep HTML clean and lightweight
- Include a proper **plain-text version**
- Avoid spam-triggering words, excessive capitalization, and too many links/images
- Maintain a healthy sender list and avoid sending to invalid or unengaged emails
- Include a clear **unsubscribe option**
- Test subject lines and content before sending
- Gradually warm up a new sending domain or IP if needed
